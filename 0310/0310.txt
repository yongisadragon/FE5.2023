div전체덩어리로 잡자. 시각적 덩어리.

-용어를 잘 기억하자. 이와같은 셀렉터는 타입셀렉터이다. (타입 셀렉터가 많으면 불안요소다. 클래스 셀렉터 등 안정적인 작업을 위해 이름을 지어준다.)

<div class(attribute)=""></div>

div {
	color(property):orange(value);
} 
attribute와 property는 한글로 둘다 속성으로 번역되지만.. 영어로 구분하도록 하자.


-property라는것은 내가 셋팅하지 않아도 기본으로 세팅된 value가 있다. 예를 들면
div {
	width: auto; (해당 요소의 '직계 부모가 제공해주는 컨텐츠 영역(containing block)'만큼 가득 채우게 하겠다는 의미이다. auto와 100%는 다르다. 그 상태에서 margin-left 100%를 추가로 넣어보면 그 이류를 알 수 있다. ) 
	background-color: transparetnt;
(지정하지 않았을 경우에는 이 값들(width,bg.c등)은 initial value*(이니셜 벨류, 초기값, default)로 설정되어 있다. 이것을 인지하는 것은 매우 중요하다. mdn에 각 속성 초기 값이 잘 설명되어 있다.)
} 

-div과 같이 블록레벨요소에 width에 고정값(px등)을 주면 오른 공간이 비는데(bg.c이 있다고 생각하고) 그 공간을 '사용 가능한 영역'이라고 한다. 이때 마진 레프트를 auto로 지정하면 그 div가 오른쪽으로 민다. 이것은 사용 가능한 공간을 마진 값을 자동으로 취한다..라고 한다. right도 auto 라고하면 가운데 정렬이 되는데, 이것은 단축속성으로 margin: 0(상하단) auto;라고 한다.
margin-left: auto; + marginl-right: auto; = margin: 0(상하단) auto(왼,오);

-class의  선택자  .class 의 앞에서 애스터리스크(universal selector)가 생략돼있다.

-height값은 auto가 기본 값이고, 기본적으로 고정값이 지정되지 않는게 좋다. 유지보수의 이유(텍스트나 컨텐츠가 엄청나게 늘어날 상황을고려해서.)그러나 고정값이 고려되어야 하는 상황이 있지만,,, 나중에 생각하자. height값은 참자.

-컨텐츠를 주르륵 넣다보면.. h1이나.. p등을 입력하다보면 위아래가 기본적으로 셋팅되는 어떤 값(마진 등)이 있다.. 이를 인스펙터에서 보면 사용자 에이전트 시트(브라우저).. 라고 나와있는데 이는 브라우저가 각각의 타입(요소)에 제공하는 기본 값이다. 크롬 기본 제공 스타일.. 이런것인데, 그래서 브라우저마다 다르게 제공된다. (헷갈리지말자.이니셜 벨류라는것은 프로퍼티에 속하는 내용이다.) 

이런건 내가 작성한 지정스타일이 아니기 때문에 나중에는 나의 디자인에 방해가 된다. 보통은 이런건 지우는데 ..통상적으로 최상단에
h1, p{
margin: initial(0);
}  
등으로..이건 margin의 이니셜벨류가 0이기 때문. 이건 margin: initial;로 명시적으로 쓸 수도 있다.(브라우저가 제공한 마진 값을 initial의 값으로 마진을 초기화해라.) 이런걸 깔끔하게 정리해야 다음 스텝이 깔끔하게 보인다.

-text-align은 다른 말로 inline elements들을 정렬한다는 의미이다. 상위 클래스에서 wrap의 text-align: center;에 의해 의해 h1이 가운데 정렬됐는가? no
text-align: center;을 하면 h1도 영향을 받는다고 생각하는데, h1 내부의 '인라인 요소=입력된 텍스트'가 가운데 정렬되는 것이다. 백그라운드나 박스 쉐도우나 보더를 맥여보자. 확인해봐. 항상

-기본 color가 black인 이유. html의 기본 칼라요소가 검정.. 이기 때문 부모의 컬러에 값을 지정하면 자동으로 아래로 전파가 된다(상속).. 인스펙터를 확인해봐도 ingerited from
h1의 스타일이 만들어지기위해 wrapper로 부터 text-align이 상속되고, header로 부터 컬러가 상속되는것.(header에 컬러 옮긴이후의 이야기) 상속이 지원되고 안되는 프로퍼티들이 있다(예를 들어 width, height 등이 상속되면 큰일남. 강제로는 상속받을 하위 자식에게 width: inherit;로 지정.. 이 역시 각 프로퍼티마다 상속 여부는 mdn에 다 나와있음). 그러나

처음 설명한 width 100%는 부모 컨텐츠 넓이에 100% 맞추는거라고 했는데, 상속의 개념은 아니다. 가져오는 거라고 하는데.
